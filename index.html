Bird's-eye view

Communication layer

  • Also known as the network layer
  • Input: a URI, output: a stream of bytes
  • Mostly consisting of
    • A URI parser
    • Protocol implementations (HTTP, FTP, etc.)
    • A local file system access layer
    • Encryption facilities
    • Caches

Processing layer

  • Input: a stream of bytes and some metadata, output: some form of presentation (hint: not always a rendering on the screen)
  • Another input source comes from the presentation layer (e.g., mouse & keyboard input)

Parsing HTML

  • Buzzword here: HTML5
  • Everything is HTML5
  • No malformed documents
  • Handling older content: quirks mode

Document Object Model (DOM)

  • HTML documents = Tree structures
  • Example: 
<html><body><div>hello</div>world</body></html>

Interesting stuff in parsing HTML

  • Incremental parsing
  • Speculative parsing
  • Representation in memory
  • Sublanguages (SVG, MathML)

Cascading Style Sheets (CSS)

  • Common CSS properties: color, font, border, etc.
  • Example: 
    div {
  color: red;
}

body > p.text {
  font: serif;
  border: thick solid blue;
}
  • Result of parsing: a set of style rules

Layout

  • Determining the dimensions of the boxes
  • Determining their presentation properties

Making sense of CSS

  • Determining the style rules applying to DOM nodes
    • Selector matching
    • Style resolution

Data structures for layout

  • Typical data structure: tree of boxes
  • This tree is internal to the browser engine
  • Demo time!

More about the layout tree

  • Mapping between nodes in layout & DOM trees
  • Not all elements in the DOM affect the output
  • Layout tree may not be similar to the DOM tree
  • Once we finish building this tree, we know how the page will "look"

What about other resources?

  • Images (static and animated)
  • Video
  • Audio

Javascript

  • JS is a full-blown programming language
  • Different JS engines work in very different ways
    • Interpretation
    • JIT Compilation
    • Optimization
  • Highly competitive area today

How does my JS code run?

  • Parse the source code into IR
  • (Optional) Analyze the code
  • Generate runnable code
    • Bytecode
    • Native machine code (Just-in-time compilation)
  • Run it
  • Modify the generated code if needed

JS meets DOM

  • Querying information from the DOM tree
  • Modifying the DOM tree
  • Calling DOM APIs on nodes in the tree

Dynamic modifications

  • Changes to the layout tree as a result of
    • Modifications to the DOM tree
    • Modifications to the styles
    • Changes in the presentation environment
  • Recreate or update the layout tree

Presentation layer

  • Event handling
